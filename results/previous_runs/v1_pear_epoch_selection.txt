# PEAR Checkpoint Evaluation — Which Epoch for PROF-GRPO?

**Purpose:** Evaluate 3 PEAR SFT checkpoints (epochs 1, 2, 3) and select the best  
*RL initialization* for PROF-GRPO, based on 5 proxy metrics measured over 50 rollout steps.

> **Key insight (PEAR paper §2):** Better offline SFT ≠ better post-RL performance.  
> The goal is an initialization that *facilitates* downstream RL.

### Metrics
| # | Metric | Ideal |
|---|--------|-------|
| M1 | GRPO proxy loss (50-step mini-GRPO) | stable, low-positive |
| M2 | Reward signal quality — correct fraction across rollouts | 0.30–0.70 ("Goldilocks") |
| M3 | PEAR token-weight entropy | higher = healthier spread |
| M4 | Forward KL to base model | moderate (1e-5 – 5e-4) |

**TOO-FEW-CORRECT fallback (new):** If a checkpoint's sampling correct-fraction  
drops below 0.15, a greedy-decoding probe distinguishes a truly broken model from  
a high-entropy one that could still respond to GRPO exploration.
